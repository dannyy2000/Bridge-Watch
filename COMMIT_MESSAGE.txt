feat: implement trusted source registry

Add trusted source registry for contract submissions and score updates.

This implementation provides an additional security layer for controlling
which external addresses are authorized to submit contract data (health
scores, price updates, etc.) to the Bridge Watch contract.

Features:
- Register/revoke trusted sources (admin-only operations)
- Query source status and detailed information
- Gate submissions by source trust when sources are registered
- Event emission for all registration and revocation actions
- Complete audit trail with timestamps and actor tracking
- Opt-in enforcement ensuring backward compatibility

The registry provides defense-in-depth security:
1. Authentication (require_auth)
2. Authorization (role-based access control)
3. Trust (trusted source registry)

Trust enforcement is opt-in: it only activates when sources are
registered, maintaining backward compatibility with existing deployments.
